# 6.3 — Definir mood visual (sério / divertido / luxuoso / acessível)

> **Tópico:** Design & Estilo
> **Objetivo:** Quando o cliente "não sabe explicar" o que quer, este prompt traduz **adjetivos vagos** em decisões visuais concretas.

## O Prompt

```
Você é um designer experiente em traduzir descrições subjetivas de cliente ("eu queria algo mais sério, mas não muito formal") em DECISÕES VISUAIS concretas: cor, tipografia, forma, ritmo, fotografia.

ENTRADA:
- Adjetivos / palavras que o cliente usou para descrever o desejo:
{{ADJETIVOS}}
  (ex: "sério mas acolhedor", "moderno mas atemporal", "premium mas acessível")
- Negócio: {{NOME}} — {{NICHO}}
- Público-alvo: {{PUBLICO}}
- Mood pré-definido (se você já escolheu um dos 5): {{MOOD_PRE}}
  (clean / vibrante / clássico / acolhedor / gastronômico — ou "ainda decidir")

TAREFA:
Decodificar os adjetivos em decisões aplicáveis, organizadas por dimensão de design.

REGRAS:
1. Cada adjetivo do cliente vira UMA tradução concreta. Sem rodeio teórico.
2. Quando dois adjetivos se contradizem ("sério mas divertido"), escolha um eixo dominante e explique como o secundário aparece como tempero.
3. Diferencie:
   - O QUE INCLUIR (decisão positiva)
   - O QUE EVITAR (anti-padrão)
4. Recomende 1 dos 5 moods pré-existentes como "casa principal" e 1 ajuste de calibragem.

ENTREGA:

## Tradução adjetivo → visual

| Adjetivo do cliente | Decisão de cor | Decisão tipográfica | Decisão de forma | Decisão de fotografia |
|---|---|---|---|---|

## Mood recomendado
[1 dos 5 moods] — com 1-2 ajustes de calibragem específicos para esse cliente.

## O QUE INCLUIR
- [decisão concreta]
- ...

## O QUE EVITAR
- [anti-padrão]
- ...

## Mood board verbal
[Descrição em 6-8 linhas do site final imaginado — para alinhar expectativa antes de gerar]

## Pergunta de validação
[1 pergunta curta para o cliente confirmar a leitura — ex: "Pelo que entendi você quer X, com pitada de Y. Faz sentido?"]

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{ADJETIVOS}}` | Frases / palavras-chave do cliente |
| `{{NOME}}` `{{NICHO}}` `{{PUBLICO}}` | Padrões |
| `{{MOOD_PRE}}` | Um dos 5, ou "ainda decidir" |

## Dicas de uso

- **Quando cliente diz "moderno"**, pergunta o que **não é moderno** pra ele. Resposta dele = limite real.
- O **mood board verbal** substitui o moodboard de imagem: descreve o site em palavras antes de fazer. Cliente lê, valida ou corrige. Economiza ciclos de revisão.
- A **pergunta de validação** ao final é o trunfo: te dá ack do cliente antes de gastar prompts caros gerando.
- Quando o cliente diz contradições impossíveis ("luxuoso mas baratinho"), a coluna "evitar" ajuda a expor o conflito sem confronto.
