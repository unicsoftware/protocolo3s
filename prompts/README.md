# 📚 Biblioteca de Prompts — Protocolo 3s

53 prompts organizados na ordem natural de um projeto: **Descoberta → Planejamento → Geração → Refino → Entrega**.

Cada arquivo segue o mesmo formato:
- **Quando usar** — contexto exato em que o prompt brilha
- **Como usar** — passo a passo de aplicação
- **O Prompt** — bloco copy-paste pronto
- **Variáveis** — tabela do que preencher
- **Dicas de uso** — pegadinhas e boas práticas

Os outputs encadeiam: o resultado de um prompt é input do próximo. A ordem importa.

---

## Fluxo recomendado

```
1. Descoberta & Briefing       (1.1 → 1.4)
       ↓
2. Estrutura & Arquitetura     (2.1 → 2.5)

7. Atalho por nicho            (substitui 3 quando aplicável)
       ↓
8. Ajustes
       ↓
9. SEO local
       ↓
10. Publicação & entrega
```

## Princípios usados na construção dos prompts

Cada prompt foi escrito aplicando estes princípios de prompt engineering:

1. **Papel claro no início** — "Você é um especialista em X" (não "act as", em português natural).
2. **Contexto antes da tarefa** — variáveis preenchidas antes da instrução.
3. **Restrições explícitas** — limites de tamanho, formato, antipadrões.
4. **Formato de saída especificado** — tabelas, seções, ordem fixa.
5. **Anti-padrões nomeados** — listar o que NÃO fazer reduz erros mais que listar o que fazer.
6. **Exemplos curtos** — quando a ambiguidade é alta.
7. **Output encadeável** — saída de um prompt vira input do próximo.
8. **Compliance embutido** — cada nicho regulamentado (OAB/CFP/CRM/CRC/CRMV/CRECI/CFO/CRO/CONFEF) tem suas regras explicitadas no prompt do Tópico 7.

## Convenções

- Variáveis usam `{{MAIUSCULA}}` — fácil de identificar e substituir.
- Cada prompt é **autossuficiente**: pode ser usado sozinho, sem ler os outros.
- A ordem dos tópicos reflete o fluxo de um projeto real — quem está perdido encontra pelo *momento* do projeto.
