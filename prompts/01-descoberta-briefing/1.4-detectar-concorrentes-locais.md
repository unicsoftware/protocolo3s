# 1.4 — Detectar concorrentes locais e pegar referências

> **Tópico:** Descoberta & Briefing
> **Objetivo:** Mapear os **3-5 principais concorrentes locais** do cliente, identificar o que cada um faz bem (e mal) no site, e extrair **referências utilizáveis** sem copiar.

---

## Quando usar

Antes de gerar o site, para responder duas perguntas: **(a)** qual o "padrão da praça" no nicho dele naquela cidade — para o site dele não ficar abaixo, e **(b)** o que dá para fazer melhor com pouco esforço.

## Como usar

1. Procure no Google: `{nicho} em {cidade}` e `{nicho} {bairro}`.
2. Anote 3-5 concorrentes com site (não só Instagram).
3. Para cada um, copie: URL, headline (texto grande do topo), 3 serviços listados, formato de contato (form, WhatsApp, telefone).
4. Cole tudo no prompt. Saída: comparativo + 5 referências aproveitáveis + 3 lacunas que viram diferencial.

## O Prompt

```
Você é um analista de concorrência focado em sites de pequenos negócios locais. Seu trabalho é olhar 3 a 5 concorrentes do cliente e extrair sinais úteis para construir um site MELHOR — não igual.

CLIENTE:
- Nicho: {{NICHO}}
- Cidade: {{CIDADE}}
- Posicionamento desejado: {{POSICIONAMENTO}}

CONCORRENTES OBSERVADOS:

{{CONCORRENTES}}

(formato esperado para cada um:
NOME — URL
HEADLINE: ...
SERVIÇOS: ...
CONTATO: ...
OBSERVAÇÃO: ...)

SUA TAREFA:
Produzir uma análise enxuta e acionável. Sem encheção. Foco no que serve para construir o site do cliente ainda hoje.

REGRAS:
1. Para cada concorrente, dê NOTA de 0 a 10 em três eixos: clareza da proposta, facilidade de contato, qualidade visual aparente. Justifique em uma linha cada.
2. Identifique o "padrão da praça": o que TODOS fazem (que vira mínimo obrigatório) e o que NINGUÉM faz (que vira diferencial barato).
3. Liste 5 elementos visuais ou de copy aproveitáveis — descreva o ELEMENTO (ex: "headline focada em medo de perder o dente"), não o concorrente, para você adaptar sem copiar.
4. Identifique 3 lacunas que o site do cliente pode preencher para se destacar.
5. Não recomende mudanças no negócio dele. Só recomende decisões de site.

FORMATO DE SAÍDA:

## 1. Comparativo rápido
| Concorrente | Clareza | Contato | Visual | Pior pecado | Maior acerto |
|---|---|---|---|---|---|

## 2. Padrão da praça
- Mínimo obrigatório (todos fazem):
- Espaço aberto (ninguém faz bem):

## 3. Referências aproveitáveis (5 elementos)
[lista numerada — descreva o elemento e como adaptar]

## 4. Lacunas que viram diferencial
[3 itens — o que o cliente pode oferecer no site que destaca dele]

## 5. Recomendação de posicionamento de site
[em uma frase: como o site do cliente deve se posicionar contra esse cenário]

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NICHO}}` | Nicho do cliente |
| `{{CIDADE}}` | Cidade / região |
| `{{POSICIONAMENTO}}` | Como ele quer ser visto (premium, popular, técnico, acolhedor) |
| `{{CONCORRENTES}}` | Bloco com 3-5 concorrentes no formato indicado |

## Dicas de uso

- **Não pule o "OBSERVAÇÃO"** de cada concorrente — escreva ali o que te chamou atenção (carregamento lento, foto ruim, sem WhatsApp). Vira nota direto.
- Se a cidade for pequena e tiver só 1-2 concorrentes com site, **inclua um concorrente de cidade vizinha do mesmo nicho** — eleva a régua.
- A seção **3 (Referências aproveitáveis)** é o melhor input para os prompts de Componentes (Tópico 4) e Copy (Tópico 5).
- Não confunda "referência aproveitável" com plágio. Você está extraindo **padrão**, não texto.
