# 10.1 — Checklist final antes de subir pro Vercel

> **Tópico:** Publicação & Entrega
> **Objetivo:** Auditoria automatizada do site **antes** de publicar — verifica responsividade, links, formulário, performance, SEO básico.

## O Prompt

```
Você é um auditor pré-publicação. Sua tarefa é rodar uma checagem ESTRUTURADA do site antes de subir, identificar problemas, e gerar um relatório acionável.

ENTRADA:
- URL do preview (Vercel preview, Lovable preview, ou local): {{URL_PREVIEW}}
- Stack: {{STACK}}
- Plataforma de deploy alvo: {{DEPLOY_TARGET}} (Vercel / Netlify / Lovable hosted / outro)
- Código fonte (se análise estática for viável): {{CODIGO}}

CHECKLIST OBRIGATÓRIO — analise e marque ✅ ou ❌ em cada item:

## 1. Responsividade
- [ ] Mobile 320px: sem overflow horizontal
- [ ] Mobile 375px: hero legível, CTA visível acima da dobra
- [ ] Tablet 768px: grids reorganizam corretamente
- [ ] Desktop 1280px+: max-width respeitado, sem texto esticando demais
- [ ] Imagens com srcset OU object-fit: cover correto

## 2. Links e CTAs
- [ ] Todos os links de WhatsApp abrem com mensagem pré-preenchida correta
- [ ] tel: e mailto: funcionam em mobile
- [ ] Links externos têm target="_blank" rel="noopener"
- [ ] Âncoras internas (#secao) rolam suavemente
- [ ] Nenhum link "404" ou "/" placeholder

## 3. Formulário (se houver)
- [ ] Validação client-side funciona (campos obrigatórios)
- [ ] Submit aciona ação correta (WhatsApp, email, ou backend)
- [ ] Honeypot ativo
- [ ] Mensagem de sucesso aparece
- [ ] Estado de erro testado

## 4. Imagens
- [ ] Todas com alt text descritivo (não "image1.jpg")
- [ ] Tamanho otimizado (mobile <100KB, desktop <300KB cada)
- [ ] WebP ou AVIF onde possível
- [ ] loading="lazy" exceto hero
- [ ] Sem imagem quebrada (broken image icon)

## 5. SEO básico
- [ ] <title> presente, único, 50-60 caracteres
- [ ] <meta description> presente, 140-160 caracteres
- [ ] OG tags (og:title, og:description, og:image, og:url)
- [ ] favicon.ico ou apple-touch-icon
- [ ] JSON-LD LocalBusiness ou subtipo correto
- [ ] sitemap.xml gerado (se multi-página)
- [ ] robots.txt presente

## 6. Performance (verificar com Lighthouse)
- [ ] LCP < 2.5s
- [ ] CLS < 0.1
- [ ] FID/INP < 200ms
- [ ] Lighthouse Performance > 85 no mobile
- [ ] Sem console.errors em runtime
- [ ] Sem 404 em recursos (CSS, JS, fontes)

## 7. Acessibilidade
- [ ] Contraste AA mínimo em todo texto
- [ ] Navegação por teclado funciona (Tab + Enter)
- [ ] Foco visível em links e botões
- [ ] aria-labels em ícones-only
- [ ] alt em imagens (✅ acima)
- [ ] H1 único na página
- [ ] Hierarquia H1 > H2 > H3 sem pular

## 8. Conteúdo
- [ ] Sem "Lorem ipsum"
- [ ] Sem placeholder de email/telefone
- [ ] Endereço, horário, contato corretos
- [ ] CRO/OAB/CRM/CRC/CRP/CRMV/CRECI visível (se aplicável ao nicho)
- [ ] Texto sem typos óbvios (revisão rápida)

## 9. Compliance por nicho
- [ ] Saúde / advocacia / contabilidade / psicologia: regras de compliance específicas respeitadas
- [ ] Sem promessa de resultado
- [ ] Sem oferta promocional indevida (advocacia)
- [ ] Sem depoimento de paciente (psicologia)

## 10. Deploy / Vercel
- [ ] vercel.json ou next.config.js correto
- [ ] Variáveis de ambiente configuradas (se houver)
- [ ] Domínio customizado pronto (se aplicável)
- [ ] HTTPS forçado
- [ ] Redirect www → naked (ou vice-versa, consistente)

ENTREGA:

## Relatório de auditoria
Para cada item marcado ❌:
- Descrição do problema
- Severidade (CRÍTICO / IMPORTANTE / DESEJÁVEL)
- Como corrigir (em 1-3 linhas)
- Trecho de código se aplicável

## Score final
- Pontos críticos pendentes: X
- Pontos importantes pendentes: X
- Pontos desejáveis pendentes: X
- Veredito: PRONTO PARA PUBLICAR / CORRIGIR ANTES DE PUBLICAR

## Próximos passos
[Lista ordenada do que fazer ANTES de subir, em ordem de prioridade]

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{URL_PREVIEW}}` | URL acessível |
| `{{STACK}}` `{{DEPLOY_TARGET}}` | Padrões |
| `{{CODIGO}}` | Opcional — se análise estática vier ao caso |

## Dicas de uso

- **Roda este prompt 1 dia antes** de mostrar pro cliente. Dá tempo pra corrigir os críticos.
- **Lighthouse no Chrome DevTools** é grátis e bate com o que o Google usa para rankear. Roda no modo mobile.
- A maior surpresa costuma ser **imagem pesada**: site com 8MB de fotos parece bonito no preview e demora 6s pra carregar no 4G.
- **Compliance do nicho** é a parte que mais escapa. Se for advocacia ou saúde, o checklist específico do prompt do Tópico 7 entra aqui também.
- O **veredito binário** ao final ajuda a não publicar com pendência crítica.
