# 9.2 — Schema.org LocalBusiness (JSON-LD) pronto

> **Tópico:** SEO Local & Meta Tags
> **Objetivo:** Gerar o **JSON-LD Schema.org** correto para o nicho do cliente, pronto para colar no `<head>`. Aparece com horário, estrelas e mapa direto na busca.

## O Prompt

```
Você é um especialista em Schema.org JSON-LD para sites de negócios locais. Sua tarefa é gerar o snippet COMPLETO, com o subtipo correto para o nicho, pronto para colar no <head>.

DADOS:
- Nome do negócio: {{NOME}}
- Nicho: {{NICHO}}
- Subtipo schema mais específico aplicável: {{SUBTIPO}}
  (Dentist / Attorney / Restaurant / HairSalon / VeterinaryCare / ExerciseGym / RealEstateAgent / AutoRepair / AccountingService / MedicalBusiness / Psychotherapist / etc. — use o mais específico do schema.org)
- Endereço completo: {{ENDERECO}}
  (rua, número, complemento, bairro, cidade, UF, CEP)
- Telefone (formato +55XX XXXXX-XXXX): {{TELEFONE}}
- WhatsApp (mesmo formato): {{WHATSAPP}}
- E-mail: {{EMAIL}}
- URL do site: {{URL_SITE}}
- URL do logo: {{URL_LOGO}}
- URL de imagem principal (foto da fachada ou hero): {{URL_IMAGEM}}
- Coordenadas geográficas (lat, long) — opcional mas recomendado: {{COORDENADAS}}
- Horário de funcionamento (lista por dia): {{HORARIO}}
- Faixa de preço ($/$$/$$$/$$$$): {{PRECO}}
- Redes sociais (URLs completas): {{REDES_SOCIAIS}}
- CRO/OAB/CRM/CRC/CRP/CRMV/CRECI (quando aplicável): {{REGISTRO}}
- Áreas atendidas (cidades/bairros, opcional): {{AREAS_ATENDIDAS}}
- Para Restaurant: tipo de cozinha, menu URL: {{RESTAURANT_EXTRA}}
- Para Attorney/Dentist/Médico: especialidades: {{ESPECIALIDADES}}

REGRAS:
1. Use o SUBTIPO mais específico (ex: Dentist > MedicalBusiness > LocalBusiness). Quanto mais específico, melhor para o Google.
2. Preencha apenas campos que TÊM dado. Não invente.
3. openingHoursSpecification em formato schema (ex: "Mo-Fr 09:00-18:00").
4. address.streetAddress, addressLocality, addressRegion, postalCode, addressCountry separados.
5. sameAs com redes sociais e perfis de Conselho profissional (OAB, CRM, etc.) quando público.
6. priceRange em formato $-$$$$.
7. geo.GeoCoordinates com latitude e longitude (números, não strings) se disponível.
8. Para Restaurant: incluir servesCuisine, hasMenu, acceptsReservations.
9. Para profissionais regulamentados: incluir membershipNumber e member do Conselho.

ENTREGA:

## JSON-LD pronto para colar
```html
<script type="application/ld+json">
{ ... }
</script>
```
(JSON formatado, com aspas duplas, sem trailing comma)

## Onde colar
- Posição: dentro do <head>, depois das meta tags principais.
- Em React/Next.js: usar <Head> ou inserir via next/script com strategy="beforeInteractive".

## Como validar
- Abra https://validator.schema.org/ e cole o snippet.
- Resultado esperado: 0 errors, 0 warnings (warnings menores são aceitáveis).
- Em produção, testar com https://search.google.com/test/rich-results.

[Lista de extras possíveis: aggregateRating (quando tiver reviews), review, makesOffer, knowsAbout]

[Se o cliente tem mais de uma unidade, gerar 1 JSON-LD por unidade, com URL diferente — esboço de exemplo]

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| Bloco padrão | Igual aos outros |
| `{{SUBTIPO}}` | O tipo schema.org mais específico |
| `{{COORDENADAS}}` | "lat, long" — pega no Google Maps clicando no ponto |
| `{{REGISTRO}}` | Conselho profissional (se aplicável) |

## Dicas de uso

- **Sempre valida no validator.schema.org** antes de subir. JSON-LD malformado é ignorado pelo Google.
- **Para multi-página**, o JSON-LD pode ir na home com tudo, OU repetido em cada página com info comum + página-específica.
- **aggregateRating** com reviews reais boosta o snippet, mas só inclui quando os reviews são realmente do site/Google e auditáveis.
- Para **clínica multi-profissional**, usa `MedicalClinic` no topo + `Physician`/`Dentist` em arrays para cada profissional.
- **NÃO inclua `aggregateRating` inventado.** Google tira no rich results e penaliza.
