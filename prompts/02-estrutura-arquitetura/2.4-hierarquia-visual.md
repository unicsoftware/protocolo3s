# 2.4 — Hierarquia visual (o que destaca primeiro)

> **Tópico:** Estrutura & Arquitetura
> **Objetivo:** Decidir, **antes do design**, o que o olho deve ver primeiro, segundo e terceiro em cada seção. Hierarquia clara > design bonito.

---

## Quando usar

Depois de decidir as seções (2.1) e antes do prompt-mestre (3.x). Sem hierarquia definida, a IA gera 4 elementos do mesmo tamanho e o site fica plano.

## Como usar

1. Cole a lista de seções.
2. Indique a ação principal de conversão.
3. Saída: para cada seção, a hierarquia em 3 níveis (primário / secundário / terciário) com regras de tamanho, peso e cor.

## O Prompt

```
Você é um designer focado em hierarquia visual para sites de pequenos negócios. Sua tarefa NÃO é escolher cores ou fontes — é decidir o que o olho vê primeiro em cada seção.

CONTEXTO:
- Lista de seções (do topo ao rodapé):
{{LISTA_SECOES}}

- Ação principal de conversão: {{ACAO_PRINCIPAL}}
- Mood desejado: {{MOOD}}
  (ex: clean, vibrante, clássico, acolhedor, gastronômico)

SUA TAREFA:
Para cada seção, definir 3 níveis de hierarquia: PRIMÁRIO (o que grita), SECUNDÁRIO (o que explica), TERCIÁRIO (o que sustenta). E regras de aplicação.

REGRAS:
1. Em cada seção, só UM elemento pode ser primário. Se você quer destacar dois, uma das duas não é realmente importante.
2. O CTA principal é sempre primário ou secundário, nunca terciário.
3. Texto longo (parágrafo) é sempre terciário, exceto na seção "Sobre".
4. Foto / imagem: defina se é primária (ocupa 50%+ da seção) ou suporte (ilustra mas não compete).
5. Use ESCALA RELATIVA, não pixels: "2x maior que o body", "peso bold vs regular", "cor de marca vs cinza neutro".

FORMATO DE SAÍDA:

## Hierarquia por seção

Para cada seção, produza este bloco:

### Seção: [nome]
- **Primário:** [elemento + regra de tamanho/peso/cor]
- **Secundário:** [elemento + regra]
- **Terciário:** [elemento + regra]
- **CTA:** [posição e nível]

## Regras globais de hierarquia
- Tamanho de H1 (hero): [escala relativa]
- Tamanho de H2 (seção): [escala relativa]
- Tamanho de body: [base]
- Peso: [quais usam bold]
- Cor de destaque: [reservada para o quê]

## Anti-padrões a evitar
[3-5 erros típicos de hierarquia em sites locais — ex: "tudo em caixa alta", "5 elementos com tamanhos parecidos", "CTA cinza no meio do parágrafo"]

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{LISTA_SECOES}}` | Output do 2.1 |
| `{{ACAO_PRINCIPAL}}` | Mesma do 2.2 |
| `{{MOOD}}` | Um dos 5 moods do prompt-mestre (3.x): clean / vibrante / clássico / acolhedor / gastronômico |

## Dicas de uso

- O output deste prompt vira **restrição direta** do prompt-mestre. Cole na seção "DESIGN RULES" dele.
- A regra "**só UM primário por seção**" é a mais importante — evita o "tudo é destaque, nada é destaque".
- Hierarquia de **mood vibrante** tende a aumentar o contraste de cor; hierarquia **clássica** trabalha mais com tamanho e peso, menos com cor.
- Se o cliente reclamar que "ficou tudo do mesmo tamanho" — provavelmente você pulou esta etapa.
