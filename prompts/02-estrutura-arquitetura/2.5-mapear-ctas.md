# 2.5 — Mapear chamadas para ação (WhatsApp, formulário, agendamento)

> **Tópico:** Estrutura & Arquitetura
> **Objetivo:** Listar **todas** as chamadas para ação do site, definir a hierarquia entre elas, e gerar os links / mensagens pré-preenchidas prontos para colar no código.

---

## Quando usar

Depois do 2.2 (fluxo de conversão). Este aqui é o *passo de implementação*: gera os links reais (`https://wa.me/55...`), as mensagens pré-preenchidas, e os parâmetros UTM para rastreio.

## Como usar

1. Diga quais canais o cliente aceita (WhatsApp, telefone, e-mail, formulário, agendamento online).
2. Forneça os dados (número, e-mail, link de agenda).
3. Saída: tabela de CTAs prontos (HTML/markdown) + texto sugerido para cada situação.

## O Prompt

```
Você é um implementador de CTAs para sites locais. Sua tarefa é gerar links FUNCIONAIS, mensagens pré-preenchidas e parâmetros de rastreio — pronto para colar no código.

DADOS DO CLIENTE:
- Nome do negócio: {{NOME_NEGOCIO}}
- Nicho: {{NICHO}}
- WhatsApp (DDI+DDD+número, só números): {{WHATSAPP}}
- Telefone fixo (opcional): {{TELEFONE}}
- E-mail: {{EMAIL}}
- Link de agendamento online (Calendly, Google Agenda público, app próprio): {{LINK_AGENDAMENTO}}
- Endereço (se aplicável): {{ENDERECO}}
- Ação principal: {{ACAO_PRINCIPAL}}
- Tom de voz: {{TOM}}

SUA TAREFA:
Produzir o conjunto completo de CTAs do site: hierarquia, links prontos, mensagens pré-preenchidas, microcopy, e variantes para diferentes seções.

REGRAS:
1. Cada link de WhatsApp deve usar o formato https://wa.me/{numero}?text={mensagem_url_encoded}
2. Mensagens pré-preenchidas devem ser CURTAS (1-2 linhas), no tom da marca, e identificar de ONDE veio o lead (ex: "Vim pelo site, gostaria de saber sobre...").
3. Para cada CTA, indique a seção em que aparece e a microcopy do botão.
4. Adicione UTMs nos links (utm_source=site, utm_medium=botao, utm_campaign={nome_secao}) APENAS quando fizer sentido (não em mailto/tel:).
5. Se o cliente tem WhatsApp e telefone, WhatsApp é primário, telefone é fallback (não competem).
6. Inclua o link "tel:" e "mailto:" prontos.
7. Para formulário, não gere o HTML do form — só descreva os campos mínimos (3-5).

FORMATO DE SAÍDA:

## Hierarquia de CTAs
1. **Primário:** [tipo + microcopy padrão]
2. **Secundário:** [tipo + microcopy padrão]
3. **Fallback:** [tipo + onde aparece]

## Links prontos para colar

### WhatsApp — versão hero
- URL: `https://wa.me/...?text=...`
- Mensagem decodificada: "..."
- Microcopy do botão: "..."

### WhatsApp — versão pós-serviços
[mesma estrutura, mensagem ajustada para quem já viu serviços]

### WhatsApp — versão pós-FAQ
[mensagem para quem ainda tem dúvida]

### Telefone (fallback)
- URL: `tel:+55...`
- Microcopy: "..."

### E-mail (rodapé)
- URL: `mailto:...?subject=...`

### Agendamento online (se aplicável)
- URL: `{{LINK_AGENDAMENTO}}` + UTMs
- Microcopy: "..."

### Formulário (se aplicável)
- Campos mínimos: [lista]
- Microcopy do botão de envio: "..."
- Mensagem de sucesso: "..."

## Mapa de CTAs por seção
| Seção | CTA primário | CTA secundário |
|---|---|---|

## Notas de implementação
- target="_blank" em quais? [lista]
- rel="noopener" obrigatório nos externos
- Tracking sugerido (event name) para cada CTA principal

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME_NEGOCIO}}` | Nome da marca |
| `{{NICHO}}` | Nicho |
| `{{WHATSAPP}}` | Número limpo, ex: `5554999990000` |
| `{{TELEFONE}}` | Fixo, ou deixe vazio |
| `{{EMAIL}}` | E-mail de contato |
| `{{LINK_AGENDAMENTO}}` | URL completa, ou vazio |
| `{{ENDERECO}}` | Endereço completo, ou vazio |
| `{{ACAO_PRINCIPAL}}` | A mesma definida no 2.2 |
| `{{TOM}}` | Tom da marca (clean, casual, formal, acolhedor) |

## Dicas de uso

- **Sempre teste o link de WhatsApp no celular** antes de entregar. Encoding errado quebra a mensagem.
- Os UTMs te dão dado depois — instala um analytics simples (Plausible, GA4) e tu vê de qual botão veio o cliente.
- A **mensagem pré-preenchida diferenciada por seção** é o detalhe que faz o cliente perceber valor: ele recebe no WhatsApp já com contexto.
- Output vira **anexo do prompt-mestre** na seção "INTEGRAÇÕES E LINKS".
