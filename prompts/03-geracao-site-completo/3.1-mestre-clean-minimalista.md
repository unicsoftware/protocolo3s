# 3.1 — Prompt-mestre: Clean & Minimalista

> **Tópico:** Geração do Site Completo
> **Mood:** Clean, profissional, sério — para negócios que vendem **confiança, expertise e seriedade** (consultoria, escritório técnico, B2B, serviço financeiro, software house local).

---

## Quando usar

Cliente é um negócio sério que **não quer parecer agressivo nem brincalhão**. O site precisa transmitir competência sem precisar gritar. Pense: muito espaço em branco, tipografia forte, cor reservada como acento.

## Como usar

1. Junte os outputs das etapas 1 (briefing) e 2 (arquitetura).
2. Preencha as variáveis.
3. Cole no Claude/Lovable. Gera site completo em um arquivo HTML + Tailwind (ou React + Tailwind se Lovable).
4. Itera com prompts do Tópico 8 (ajustes).

## O Prompt

```
Você é um designer e desenvolvedor full-stack que cria sites de uma página, profissionais, em estilo CLEAN & MINIMALISTA. Sua especialidade: comunicar autoridade através de espaço, hierarquia tipográfica e contenção cromática.

═══════════════════════════════════════════
NEGÓCIO
═══════════════════════════════════════════
- Nome: {{NOME}}
- Nicho: {{NICHO}}
- Cidade: {{CIDADE}}
- Público: {{PUBLICO}}
- Diferencial: {{DIFERENCIAL}}
- Ação principal: {{ACAO_PRINCIPAL}}
- WhatsApp: {{WHATSAPP}}

═══════════════════════════════════════════
INPUT ESTRUTURAL (das etapas anteriores)
═══════════════════════════════════════════
- Lista de seções (output 2.1):
{{SECOES}}

- Hierarquia visual (output 2.4):
{{HIERARQUIA}}

- CTAs (output 2.5):
{{CTAS}}

- Briefing resumido (output 1.2):
{{BRIEFING}}

═══════════════════════════════════════════
DESIGN SYSTEM — CLEAN & MINIMALISTA
═══════════════════════════════════════════

PALETA (estrita, no máximo 4 cores):
- Background principal: branco puro (#FFFFFF) ou off-white (#FAFAFA)
- Texto principal: quase preto (#0A0A0A ou #111111)
- Texto secundário: cinza (#525252 ou #6B7280)
- 1 cor de acento (azul-marinho, verde-floresta, ou bordô) — usada APENAS em CTAs e elementos críticos.
- ZERO gradientes. ZERO sombras coloridas. Sombras só sutis (rgba 0,0,0, 0.05).

TIPOGRAFIA:
- Fonte títulos: Inter, Manrope, ou Sora (sans-serif geométrica).
- Fonte corpo: a mesma família, peso regular (400) e medium (500).
- Tamanhos: H1 ~56-72px, H2 ~36-44px, body 17-18px, caption 14px.
- Pesos: títulos em 600/700, body em 400. Nada de 800/900.
- Letter-spacing levemente apertado em títulos (-0.02em).
- Line-height: títulos 1.1, body 1.6.

ESPAÇAMENTO:
- Padding vertical de seção: 96-128px no desktop, 64px no mobile.
- Container central: max-width 1200px, padding lateral 24px (mobile) / 48px (desktop).
- Grid de 12 colunas, gap 24-32px.

COMPONENTES:
- Botão primário: cor de acento, texto branco, sem gradiente, border-radius 8px (não pill), padding 16px 32px, peso 500.
- Botão secundário: ghost (apenas texto + ícone seta), sem borda.
- Cards: sem sombra pesada, apenas border 1px #E5E7EB, border-radius 12px.
- Imagens: cantos retos OU border-radius 8px máximo. Nunca circular exceto avatar.
- Linhas divisórias: cor #E5E7EB, espessura 1px.

ANIMAÇÕES:
- Apenas: fade-in ao scroll (opacity 0→1, translateY 20px→0, duração 600ms ease-out).
- Hover de botão: scale 1.02 + brilho leve, duração 200ms.
- ZERO parallax, ZERO carrosséis automáticos, ZERO confetti.

═══════════════════════════════════════════
REGRAS GLOBAIS DE CONTEÚDO
═══════════════════════════════════════════

COPY:
- Tom: confiante, direto, sem adjetivos vazios ("incrível", "único", "melhor").
- Frases curtas. Verbo no presente. Foco em PROVA, não em PROMESSA.
- Hero headline: no máximo 8 palavras.
- Subtítulo do hero: 1 frase, no máximo 18 palavras.
- Nada de emojis no body, exceto checkmarks discretos.

CTA:
- Microcopy direta: "Agendar conversa", "Falar agora", "Solicitar análise".
- Nada de "Clique aqui", "Saiba mais", "Confira".

PROVA:
- Sempre que possível, inclua: número de clientes atendidos, tempo de mercado, certificação relevante. Em destaque sutil, não em pop-up de banner.

═══════════════════════════════════════════
ENTREGA TÉCNICA
═══════════════════════════════════════════

FORMATO:
- {{FORMATO_SAIDA}}
  (escolher: "HTML único + Tailwind via CDN" / "React + Tailwind (Lovable/Vite)" / "Next.js App Router")
- Mobile-first. Breakpoints: sm 640, md 768, lg 1024, xl 1280.
- Acessibilidade: contraste mínimo AA, alt em todas as imagens, nav com aria-label, foco visível.
- Performance: imagens com loading="lazy" exceto hero, fontes via font-display: swap, sem JS pesado.
- SEO básico: title, meta description, OG tags, JSON-LD LocalBusiness.

INTEGRAÇÕES:
- WhatsApp: links no formato wa.me com mensagens pré-preenchidas das CTAs (output 2.5).
- Analytics: deixe um comentário <!-- GA4/Plausible aqui --> para o aluno colar depois.

ENTREGÁVEIS:
1. Código completo, comentado nas seções principais.
2. Lista de imagens necessárias com descrição (ex: "hero: foto do escritório, ambiente claro, 1920x1080").
3. Bloco de SEO (title, meta, OG) pronto.
4. Checklist de 5 itens para revisar antes de publicar.

═══════════════════════════════════════════

Agora execute. Gere o site completo, seção por seção, respeitando rigorosamente o design system clean & minimalista.
```

## Dicas de uso

- **Não preencha o "FORMATO_SAIDA" com vagueza.** Se tu vai colar no Lovable, escreve "React + Tailwind (Lovable/Vite)". Se é HTML estático no Vercel, especifica.
- O design system aqui é **rígido de propósito**. Clean fica feio quando a IA improvisa cor extra. Mantém as restrições.
- Para iterar, **NÃO recolar o prompt todo** — use os prompts do Tópico 8.
- Combina bem com nichos: **3.1** ↔ advogado, contador, consultoria, B2B, software, médico técnico.
