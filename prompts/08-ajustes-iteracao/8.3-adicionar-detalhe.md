# 8.3 — Adicionar detalhe sem sobrecarregar

> **Tópico:** Ajustes & Iteração
> **Objetivo:** Adicionar 1 informação ou elemento novo **sem quebrar a hierarquia** existente.

## O Prompt

```
Você é um designer cuidadoso. Sua tarefa é encaixar UM elemento novo numa seção existente — preservando hierarquia, ritmo e equilíbrio.

SEÇÃO ATUAL:
{{CODIGO_SECAO}}

ELEMENTO A ADICIONAR:
{{ELEMENTO}}
(ex: "selo de certificação ISO", "novo serviço", "frase de oferta", "logo de prêmio recebido")

JUSTIFICATIVA DO PORQUÊ ADICIONAR:
{{JUSTIFICATIVA}}
(ex: "cliente acabou de receber prêmio", "promoção do mês", "selo de regulamentação")

PRIORIDADE NA HIERARQUIA:
{{PRIORIDADE}}
(primária / secundária / terciária — secundária ou terciária é o normal para adições)

REGRAS:
1. Identifique se REALMENTE cabe na seção atual OU se merece outro lugar. Se a seção já está cheia, indique seção alternativa.
2. Adicione no nível de hierarquia indicado:
   - Primária: rara, e implica REORDENAR.
   - Secundária: ao lado/abaixo do destaque primário, em escala visual menor.
   - Terciária: como linha de apoio, badge, footnote, ou logo discreto.
3. NÃO inflar tudo: o elemento existente fica do mesmo tamanho. O novo tem que CABER.
4. Use ESCALA RELATIVA (menor que o primário, próximo ao secundário) — não fixa.
5. Se for badge/selo, define formato discreto (pill, ícone+texto, badge canto).
6. Se for promoção/oferta, considere zona de destaque adequada (perto do CTA, não no header).
7. Avalie acessibilidade: cor contrastante, alt em imagens, semântica.

ENTREGA:
1. Decisão: "elemento entra em [posição] como [tipo visual] em escala [pequena/média]".
2. Código atualizado da seção.
3. Justificativa visual: por que entrou ali e não em outro lugar.
4. Alerta se houver risco: "este elemento compete com X — considere remover X ou mover para Z".

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{CODIGO_SECAO}}` | Seção atual |
| `{{ELEMENTO}}` | O novo elemento (texto, badge, selo, link) |
| `{{JUSTIFICATIVA}}` | Por que adicionar |
| `{{PRIORIDADE}}` | Hierarquia desejada |

## Dicas de uso

- Quando o cliente pede para "**adicionar mais coisa**", primeiro pergunte: **isso é primário, secundário ou terciário?** A resposta dele te diz onde encaixar.
- Quase tudo que cliente quer adicionar é **terciário** — ele só quer estar visível, não dominar.
- Se a seção já está cheia, o output deve ser claro: "**não cabe aqui — sugiro mover para X**". Empilhar é a tentação errada.
- Para **selo / certificação / prêmio**, padroniza posição: rodapé OU faixa de "confiança" perto do hero. Não espalhar.
