# 8.1 — Alteração isolada sem mexer no resto

> **Tópico:** Ajustes & Iteração
> **Objetivo:** Pedir uma mudança PONTUAL sem que a IA "aproveite" e refaça meio site.

## Quando usar

Cliente pediu "muda só o título do hero pra X" — e a IA, animada, reescreveu sobre, depoimentos e mexeu em cores. Este prompt previne isso.

## O Prompt

```
Você é um desenvolvedor cirúrgico. Sua tarefa é fazer EXATAMENTE a alteração pedida — nem mais, nem menos. Cada palavra, cor, espaço fora do escopo permanece intocado.

ARQUIVO / CÓDIGO ATUAL:
{{CODIGO_ATUAL}}

ALTERAÇÃO SOLICITADA:
{{ALTERACAO}}

REGRAS RÍGIDAS:
1. Aplique APENAS a alteração descrita.
2. NÃO refaça outras seções "para combinar".
3. NÃO mude classes, estrutura, ordem, IDs, props ou nomes de variáveis fora do escopo.
4. NÃO "melhore" copy, espaçamento, cor, tipografia que não foi pedido.
5. NÃO adicione comentários explicativos no código alterado (a menos que o pedido contenha código novo significativo).
6. Se a alteração afeta legitimamente outro lugar (ex: mudou nome do botão e há tracking que cita o nome), liste essas dependências em uma seção separada — mas NÃO altere automaticamente. Pergunta antes.
7. Se a alteração for ambígua, faça uma pergunta SIM/NÃO específica antes de codar.

ENTREGA:
1. Diff conceitual: "antes → depois" do trecho exato modificado.
2. Código atualizado COMPLETO da seção/arquivo afetado (não só o trecho), para o aluno colar de volta.
3. Lista de "Dependências detectadas que NÃO alterei" — opcional, só se houver.
4. Confirmação no fim: "Alterei apenas X. Tudo o mais está como estava."

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{CODIGO_ATUAL}}` | Cole o código atual (arquivo inteiro ou seção relevante) |
| `{{ALTERACAO}}` | Descreva o que mudar — específico |

## Dicas de uso

- **Funciona melhor com escopo MENOR.** Se a alteração toca 5 lugares, divide em 5 chamadas.
- A regra "**NÃO melhore o que não foi pedido**" é o coração do prompt. Sem ela, IA gosta de ser útil demais.
- Se o cliente diz "**altera o site todo pra ficar mais elegante**", isso NÃO é alteração isolada — usa o **prompt-mestre 3.x** de novo, com novo brief.
- **Diff conceitual antes do código** ajuda a auditar rápido se a IA respeitou o escopo.
