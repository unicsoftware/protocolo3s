# 4.4 — Galeria de fotos (com lightbox)

> **Tópico:** Componentes & Seções
> **Objetivo:** Mostrar trabalho realizado, ambiente do estabelecimento, ou portfolio em uma galeria **performática, com lightbox** (clique abre versão grande).

---

## Quando usar

Negócios que vendem com a imagem: salão (cabelos), barbearia, dentista (sorrisos), academia (transformações), restaurante (pratos e ambiente), arquiteto, fotógrafo, pet shop (banho/tosa), tatuador.

## O Prompt

```
Você é um designer e dev front-end especializado em galerias de fotos para sites de pequenos negócios. Sua tarefa é entregar UMA galeria — performática, responsiva, com lightbox limpo.

CONTEXTO:
- Negócio: {{NOME}} — {{NICHO}}
- Tipo de galeria: {{TIPO}}
  (portfolio / antes-e-depois / ambiente / produtos / mosaico geral)
- Quantidade de fotos: {{QTD_FOTOS}}
- Categorias / agrupamentos: {{CATEGORIAS}}
  (ex: "cortes / coloração / penteado de noiva" — ou vazio se não tem)
- Mood do site: {{MOOD}}
- Stack: {{STACK}}

REGRAS:
1. Layout grid:
   - 6 fotos: 3 colunas, 2 linhas
   - 8-12 fotos: 4 colunas
   - 12+: 3-4 colunas com ALTURA VARIADA (masonry leve) para criar ritmo
2. Aspect-ratio CONSISTENTE dentro da mesma fileira. Pode variar entre fileiras se for masonry.
3. Cada foto tem hover: leve scale (1.03) + overlay com nome/categoria opcional.
4. Click abre LIGHTBOX:
   - Fundo escurecido (rgba 0,0,0,0.85).
   - Foto grande centralizada, max-h 90vh.
   - Setas de navegação esquerda/direita (teclado também: ← →).
   - Botão fechar (X) + clique fora fecha + ESC fecha.
   - Indicador "3 / 12" no canto.
5. Filtros por categoria (se houver categorias): pills no topo, filtro client-side, transição suave.
6. Performance:
   - loading="lazy" em todas as fotos exceto as 4 primeiras.
   - srcset com 2-3 tamanhos.
   - Use thumbnails na grid (max 600px de largura). Versão full só carrega no lightbox.
7. Mobile: 2 colunas, sem masonry, lightbox em fullscreen.
8. Antes/depois (se TIPO = antes-e-depois):
   - Slider duplo dentro do card (drag horizontal revela).
   - OU duas fotos lado a lado com label "Antes" / "Depois".
   - Lightbox abre com o slider funcional.

ANTI-PADRÕES:
- Lightbox que não fecha com ESC.
- Carregar 30 fotos em alta resolução de uma vez.
- Foto sem alt text.
- Layout que quebra em mobile (overflow horizontal).

ENTREGA:
1. HTML/JSX da seção completa, incluindo:
   - Grid de thumbs.
   - Filtros (se aplicável).
   - Lightbox modal (component).
   - Lógica de teclado e clique-fora.
2. Use biblioteca leve se for React: yet-another-react-lightbox ou photoswipe. Em HTML puro, escreva o lightbox mínimo.
3. Lista de fotos placeholder com {{QTD_FOTOS}} entradas, com alt sugerido por categoria.
4. CSS necessário (Tailwind classes ou bloco style).

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME}}` `{{NICHO}}` | Básicos |
| `{{TIPO}}` | Tipo de galeria |
| `{{QTD_FOTOS}}` | Número aproximado |
| `{{CATEGORIAS}}` | Lista separada por `/`, ou "nenhuma" |
| `{{MOOD}}` `{{STACK}}` | Padrões |

## Dicas de uso

- **Sempre teste o lightbox no celular real** — bibliotecas mal configuradas dão problema de scroll.
- Para **antes/depois de salão/dentista**, peça consentimento por escrito do cliente antes de publicar foto reconhecível.
- Se o cliente tem **menos de 6 fotos boas**, não monta galeria. Substitui por uma seção de "destaques" 2-3 fotos. Galeria pobre denigre o site.
- **Otimize as imagens** antes de subir (TinyPNG, Squoosh) — site de barbearia com 8MB de fotos é cliente perdido.
