# 4.7 — FAQ com accordion

> **Tópico:** Componentes & Seções
> **Objetivo:** Responder as **objeções reais** do visitante antes que ele clique fora — em formato accordion, escaneável, com 5-9 perguntas.

---

## Quando usar

Sempre que houver dúvidas frequentes que **bloqueiam a conversão** (preço, agendamento, documentação, garantia, demora, processo). FAQ não é decorativo — é remoção de fricção.

## O Prompt

```
Você é um especialista em remoção de objeções para sites de pequenos negócios locais. FAQ não é "perguntinhas curiosas" — é a lista das DÚVIDAS QUE TRAVAM o cliente de comprar.

CONTEXTO:
- Negócio: {{NOME}} — {{NICHO}}
- Cidade: {{CIDADE}}
- Lista das principais objeções/dúvidas conhecidas (do briefing 1.x ou do conhecimento do cliente):
{{OBJECOES}}
- Política de preços / pagamento / desconto / convênio: {{POLITICA}}
- Modalidade de atendimento: {{MODALIDADE}}
- Mood do site: {{MOOD}}
- Stack: {{STACK}}

REGRAS:
1. 5 a 9 perguntas. Menos = não cobre. Mais = ninguém lê.
2. Pergunta DEVE estar na voz do cliente, não na voz do negócio.
   - ✅ "Quanto custa uma consulta?"
   - ❌ "Quais são os valores praticados pela clínica?"
3. Resposta tem 30-80 palavras. Direto. Sem rodeio. Sem "como mencionado anteriormente".
4. Se a resposta exige nuance (ex: preço varia), DÊ A FAIXA, não escapa: "Consulta a partir de R$X. Avaliação inicial é gratuita."
5. Cobrir as 5 categorias clássicas de objeção (escolher as relevantes ao nicho):
   - PREÇO ("quanto custa?", "tem desconto?", "aceita parcelar?")
   - PROCESSO ("como funciona?", "quanto tempo demora?")
   - QUALIFICAÇÃO ("quem atende?", "tem registro?")
   - GARANTIA ("e se eu não gostar?", "tem garantia?")
   - LOGÍSTICA ("onde fica?", "tem estacionamento?", "atende fora do horário?")
6. Accordion:
   - Apenas 1 aberto por vez (sanfona) — exceto se o nicho exige múltiplas abertas.
   - Animação suave (200-300ms ease).
   - Ícone +/− ou seta rotacionada à direita.
   - Pergunta clicável em toda a largura (não só o ícone).
   - Estado aberto destacado visualmente (cor de fundo levemente diferente OU borda).
7. Acessibilidade:
   - <button> com aria-expanded.
   - aria-controls apontando para o painel.
   - Painel com role="region" e aria-labelledby.
   - Navegável por teclado (Enter/Space abre).
8. SEO bonus: incluir JSON-LD FAQPage para aparecer com rich snippet no Google.

ANTI-PADRÕES:
- "Quem somos?" como FAQ — não é dúvida, é seção sobre.
- Resposta de 1 linha vazia ("Sim, atendemos") sem contexto.
- Resposta de 3 parágrafos.
- Pergunta retórica ("Por que escolher a {nome}?") — disfarce de marketing.

SE A LISTA DE OBJEÇÕES VEIO VAZIA:
- Use o nicho para inferir as 5-7 objeções mais frequentes.
- Marque cada item gerado com [INFERIDO — confirmar com cliente].

SAÍDA:
1. HTML/JSX completo do accordion, na stack indicada.
2. Lista final das perguntas com respostas redigidas.
3. JSON-LD FAQPage pronto para colar no <head>.
4. Sugestão de headline da seção (ex: "Antes de marcar, talvez você queira saber").

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME}}` `{{NICHO}}` `{{CIDADE}}` | Básicos |
| `{{OBJECOES}}` | Lista de dúvidas reais (do briefing) |
| `{{POLITICA}}` | Como cobra, parcela, aceita convênio |
| `{{MODALIDADE}}` | Presencial / online / híbrido |
| `{{MOOD}}` `{{STACK}}` | Padrões |

## Dicas de uso

- **A pergunta mais importante é a de PREÇO.** Se o cliente "não quer falar de preço no site", pelo menos diga uma faixa ou "consulta de avaliação gratuita". Esconder espanta.
- O **JSON-LD FAQPage** te dá chance de aparecer no Google em formato expandido. Vale o esforço.
- Para nicho **médico/jurídico**, cuidado com promessas: "Tem garantia?" → resposta deve ser ética, não comercial agressiva.
- Se o cliente **não conhece as objeções**, pergunta no WhatsApp: "Quais são as 5 perguntas que mais ouve antes de fechar?". Resposta dele = FAQ pronto.
