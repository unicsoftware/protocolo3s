# 4.5 — Seção de Depoimentos

> **Tópico:** Componentes & Seções
> **Objetivo:** Apresentar depoimentos de forma **crível** (não decorativa) — o visitante deve acreditar que aquilo é real.

---

## Quando usar

Quase todo site precisa. Mas só inclua se o cliente **tiver depoimentos reais**. Depoimento inventado destrói credibilidade quando descoberto.

## O Prompt

```
Você é um especialista em prova social para sites locais. Seu trabalho NÃO é inventar depoimentos — é apresentar os existentes de forma que pareçam reais (porque são).

CONTEXTO:
- Negócio: {{NOME}} — {{NICHO}}
- Depoimentos disponíveis (texto + nome + cargo/contexto + foto se houver):
{{DEPOIMENTOS}}
- Origem dos depoimentos: {{ORIGEM}}
  (Google Reviews / iFood / Insta DM / WhatsApp / formulário próprio)
- Mood do site: {{MOOD}}
- Stack: {{STACK}}

REGRAS:
1. Quantidade: 3 a 6 depoimentos visíveis. Mais que isso = ninguém lê. Se o cliente tem 30, escolha os 4-5 mais ESPECÍFICOS (não os mais elogiosos).
2. Layout:
   - 3 depoimentos: grid 3 colunas
   - 4-6 depoimentos: 2 colunas (mais legível)
   - Mobile: 1 coluna, sem carrossel
3. Cada card tem:
   - Aspas decorativas grandes (ornamento, não conteúdo).
   - Texto do depoimento (50-150 palavras — corte o que for excessivo).
   - Nome do cliente.
   - Contexto (cidade, profissão, há quanto tempo é cliente, ou serviço usado).
   - Foto se autorizada (avatar 48px, circular).
   - Selo de origem (mini-ícone "★ Google" / "iFood" / etc.) — gera credibilidade.
4. Estrelas: 5 estrelas amarelas/cor-de-marca. Se o original é Google Reviews, mantém o número real.
5. Carrossel: APENAS se houver 7+ depoimentos. Indicadores de página visíveis. NUNCA autoplay.
6. Headline da seção: específica, não genérica. Ex: "O que dizem quem já passou por aqui" > "Depoimentos".
7. Prova adicional: se o cliente tem nota global no Google ("4.9 ★ • 184 avaliações"), exibir em destaque NA seção, com link para a página de reviews.

ANTI-PADRÕES:
- Depoimento "Sou Maria, adorei o serviço, super recomendo!" — vazio.
- 5 depoimentos genéricos sem nome ou só com "M.S.".
- Foto de stock como "cliente".
- Carrossel automático que rola enquanto a pessoa lê.
- Inventar depoimento. Ponto.

EDIÇÃO PERMITIDA:
- Cortar partes longas, mantendo intenção.
- Corrigir gramática SEM alterar voz.
- Ajustar pontuação.
NÃO permitido:
- Reescrever para "ficar melhor".
- Mudar nome ou contexto.

SE OS DEPOIMENTOS VIEREM CRUS (texto bagunçado):
- Edite cada um para 50-100 palavras, preservando voz e fato concreto.
- Indique entre colchetes o que foi cortado [...].
- Marque cada um com [REVISAR COM CLIENTE] após edição.

SAÍDA:
1. HTML/JSX da seção, na stack indicada.
2. Lista final dos depoimentos editados com nome, contexto, origem.
3. Recomendação de quantos exibir (de quantos foram fornecidos).
4. Sugestão de microcopy de headline e subhead.
5. Se houver nota Google: bloco visual de destaque pronto para colar.

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME}}` `{{NICHO}}` | Básicos |
| `{{DEPOIMENTOS}}` | Texto bruto, separado por linha em branco entre cada |
| `{{ORIGEM}}` | De onde veio cada |
| `{{MOOD}}` `{{STACK}}` | Padrões |

## Dicas de uso

- **Pegue os depoimentos do Google Maps** copiando direto. Já vem com nome, foto e estrela.
- Se o cliente **não tem depoimento ainda**, pula a seção e bota outro elemento de prova (anos de mercado, nº atendimentos, certificação). NÃO invente.
- **Depoimento específico** ("ela tirou meu siso sem dor") > genérico ("ótimo profissional"). Filtra os mais específicos mesmo que mais curtos.
- Inclua o **link para o Google Reviews** abaixo da seção: "Ver todas as 184 avaliações no Google →".
