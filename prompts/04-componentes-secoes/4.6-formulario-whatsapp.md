# 4.6 — Formulário de contato com integração WhatsApp

> **Tópico:** Componentes & Seções
> **Objetivo:** Formulário curto que, ao enviar, **abre o WhatsApp do cliente** com a mensagem pré-preenchida — zero infraestrutura de e-mail/SMTP, zero spam.

---

## Quando usar

Quando o cliente quer "formulário" mas não tem painel/CRM/e-mail backend. Solução: form que monta uma mensagem e dispara `wa.me` no submit. Funciona 100% client-side.

## O Prompt

```
Você é um dev front-end especializado em formulários leves para sites de pequenos negócios. Sua tarefa é construir UM formulário de contato que, ao submeter, abre o WhatsApp do dono do negócio com mensagem montada a partir dos campos.

CONTEXTO:
- Negócio: {{NOME}} — {{NICHO}}
- Número WhatsApp (DDI+DDD+número, só dígitos): {{WHATSAPP}}
- Campos desejados: {{CAMPOS}}
  (default mínimo: nome, telefone/WhatsApp, mensagem. Adicione apenas o que for ÚTIL para qualificar o lead — ex: serviço de interesse, melhor horário, cidade)
- Mood do site: {{MOOD}}
- Cor de marca: {{COR_MARCA}}
- Stack: {{STACK}}

REGRAS:
1. NUNCA mais de 5 campos. Cada campo a mais reduz conversão.
2. Validação client-side leve: nome obrigatório, telefone obrigatório com máscara BR, mensagem opcional.
3. Ao submeter:
   a. Validar campos.
   b. Montar mensagem no formato:
      "Oi! Vim pelo site.
      Nome: {nome}
      WhatsApp: {telefone}
      [Serviço de interesse: {servico}]
      [Mensagem: {mensagem}]"
   c. URL-encode a mensagem.
   d. Abrir https://wa.me/{numero}?text={mensagem_encoded} em nova aba.
   e. Mostrar feedback de sucesso ("Te chamamos no WhatsApp!").
4. Sem reCAPTCHA (overkill para volume de site local).
5. Honeypot anti-spam: campo invisível "website" com display:none. Se preenchido, abortar submit silenciosamente.
6. Acessibilidade:
   - Label associado a cada input (htmlFor).
   - aria-required em obrigatórios.
   - Mensagens de erro com aria-live.
   - Foco visível em todos os campos.
7. Layout:
   - 1 coluna em mobile, 2 colunas em desktop (nome + telefone na mesma linha).
   - Botão de submit largura total no mobile.
   - Botão com cor de marca, microcopy direta ("Falar no WhatsApp", não "Enviar").
8. Estados visuais:
   - Default
   - Focused (borda na cor de marca)
   - Error (borda vermelha + texto de erro)
   - Disabled durante submit (spinner pequeno).

ANTI-PADRÕES:
- 12 campos pedindo CPF, endereço, data de nascimento.
- Botão "Enviar" cinza minúsculo no canto.
- Sem validação — manda mensagem em branco no WhatsApp.
- reCAPTCHA v3 que quebra o formulário.

ENTREGA:
1. HTML/JSX completo do formulário, na stack indicada.
2. Lógica de submit (JS vanilla ou React handler).
3. CSS / Tailwind classes.
4. Bloco de instruções para o cliente: "Como ativar / mudar número".
5. Mensagem-modelo final que será enviada ao WhatsApp (mostre exemplo preenchido).
6. Snippet de teste: como simular submit no console.

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME}}` `{{NICHO}}` | Básicos |
| `{{WHATSAPP}}` | Número limpo (ex: 5554999990000) |
| `{{CAMPOS}}` | Lista de campos desejados — ou "default" para usar o mínimo |
| `{{MOOD}}` `{{COR_MARCA}}` `{{STACK}}` | Padrões |

## Dicas de uso

- **Sempre 3-4 campos**. Nome + WhatsApp + (opcional 1 select de serviço) + (opcional 1 textarea curto). Mais que isso, conversão despenca.
- **Honeypot resolve spam** em 99% dos casos. reCAPTCHA é exagero para site local.
- Se o cliente **insistir em receber por e-mail**, use Formspree (free tier) ou Netlify Forms — adicionar como ação secundária do submit.
- **Teste o link wa.me final** com mensagem real preenchida antes de entregar. Encoding errado quebra a quebra de linha.
