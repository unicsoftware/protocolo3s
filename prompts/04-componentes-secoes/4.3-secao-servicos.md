# 4.3 — Seção de Serviços (cards com ícone + descrição)

> **Tópico:** Componentes & Seções
> **Objetivo:** Apresentar serviços de forma **escaneável**, com card por serviço (ícone + nome + descrição curta + CTA opcional).

---

## Quando usar

Sempre que o site precisa listar 3 a 9 serviços/produtos. Para listas maiores, use catálogo (multi-página) ou agrupe.

## O Prompt

```
Você é um designer de UI especializado em seções de serviços para pequenos negócios. Sua tarefa é entregar UMA seção de serviços, escaneável, com cards equilibrados.

CONTEXTO:
- Negócio: {{NOME}} — {{NICHO}}
- Lista de serviços (com nome + descrição curta do cliente, ou vazio para gerar):
{{SERVICOS}}
- Mood do site: {{MOOD}}
- Cor de marca: {{COR_MARCA}}
- CTA por card? {{CTA_POR_CARD}} (sim / não)
- Link CTA: {{LINK_CTA}}
- Stack: {{STACK}}

REGRAS:
1. Cada card tem: ÍCONE (ou ilustração leve) + NOME do serviço (1 linha) + DESCRIÇÃO (15-30 palavras) + opcional CTA.
2. Descrições têm o MESMO comprimento (variação máxima de ±5 palavras) — desalinhamento visual = bagunça.
3. Cards do MESMO tamanho. Use grid CSS, não float.
4. Quantidade de cards na linha:
   - 3 serviços: 3 colunas
   - 4 serviços: 2 ou 4 colunas (não 3)
   - 6 serviços: 3 colunas (2 linhas)
   - 5 ou 7 serviços: usar 3+2 ou 4+3 com último card centralizado
5. Ícone: line-style 2px, monocromático na cor de marca. Sem ícones cartoon coloridos. Tamanho consistente (~40px).
6. Hover: lift sutil (-2 a -4px) + sombra cresce.
7. Mobile: 1 card por linha, gap 16px.
8. Headline da seção: 4-7 palavras, foco no benefício (ex: "Tudo que sua boca precisa") — não "Nossos serviços".

ANTI-PADRÕES:
- 9 cards de tamanhos diferentes.
- Descrições com 3 frases num e 1 palavra noutro.
- Ícone diferente em cada card (ora outline ora solid ora colorido).
- Card com 5 sub-bullets — vira lista de feature, não card de serviço.

SE A LISTA DE SERVIÇOS VEIO VAZIA OU INCOMPLETA:
- Use o nicho para inferir os 5-8 serviços padrão.
- Marque cada item gerado como [SUGESTÃO — confirmar com cliente].

SAÍDA:
1. HTML/JSX completo da seção, na stack indicada.
2. Tabela final com: Nome | Descrição (final) | Ícone sugerido (nome do ícone Lucide ou Heroicons).
3. Recomendação de quantidade ideal de cards visíveis (se a lista tem 9, talvez seja melhor mostrar 6 + "ver mais").

Agora execute.
```

## Variáveis

| Variável | O que preencher |
|---|---|
| `{{NOME}}` `{{NICHO}}` | Básicos |
| `{{SERVICOS}}` | Lista (nome + descrição curta) ou vazio |
| `{{MOOD}}` `{{COR_MARCA}}` `{{STACK}}` | Padrões |
| `{{CTA_POR_CARD}}` | "sim" se cada card leva a uma página/WhatsApp específico |
| `{{LINK_CTA}}` | URL alvo (se CTA por card = sim) |

## Dicas de uso

- **Use Lucide ou Heroicons** para os ícones — biblioteca consistente, importação fácil.
- Quando o cliente listar **15 serviços**, agrupe em 4-6 categorias macro. Catálogo gigante = visitante perdido.
- A regra do **mesmo comprimento de descrição** é o detalhe que diferencia card "feito" de card "profissional".
- **CTA por card** só faz sentido se cada serviço tem uma jornada distinta (ex: hamburgueria não precisa, dentista pode ter um WhatsApp diferente para "agendar limpeza" vs "tirar siso").
